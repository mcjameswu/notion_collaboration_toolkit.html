五大工作流 · 完整提示詞 · 可重用模組 · 為 M.C. James 設計
1
WF-1｜全域啟動工作流
用於：每次新開對話、想全面推進 Notion 系統優化時
目的：建立完整共識起點 適用：任何新對話

無論距離上次對話多久，都能在 30 秒內讓新的 Claude 實例完全理解你的系統設計邏輯、待辦事項、限制條件，直接進入執行模式。

步驟流程
  • 開啟新的 Claude 對話視窗（建議用 Claude.ai Projects 功能，讓記憶持久）
  • 貼入下方「WF-1 主啟動提示詞」，等待 Claude 確認理解
  • Claude 會列出它對你系統的理解摘要，你做確認或修正
  • 告訴 Claude 今天想推進哪個工作流或具體任務
  • 根據回覆決定是否需要補充背景，或直接執行
主啟動提示詞 (WF-1-A)
複製此提示詞貼入新對話You are my dedicated Notion PKM system advisor and Life Dashboard architect. Before we start, here is the permanent context you must internalize: IDENTITY & SYSTEM GOAL: - I am M.C. James, first-time CEO of 巨明生醫 (a biomedical startup) - I am also building a second personal venture and personal brand - My life has six equal dimensions: company, family, wealth, health, learning, relationships - My PKM system must reflect my WHOLE LIFE, not just work CORE DESIGN PRINCIPLES (never violate these): 1. Friction is the enemy — every design choice must minimize activation energy 2. External files (PPT, Excel, paper) are reality — index and connect them, never demand migration 3. The system must surface a whole-life overview on open, not just professional tasks 4. Capture happens but review never does — any design must build in a minimal review mechanism 5. Structural confusion causes avoidance — clear "where does this go" rules are mandatory KNOWN PAIN POINTS (in priority order): 1. Structural confusion — unclear where to record information 2. Notes go unreviewed after capture 3. Disconnected external files with no bridge to Notion 4. No life-overview visible on opening Notion 5. Psychological burden — the system feels like a liability CURRENT PLATFORM: Notion (primary PKM) CURRENT PROJECT: Notion system rescue and redesign COLLABORATION RULES: - Always confirm your understanding of context before executing changes - Propose before acting — tell me the plan, wait for approval - When in doubt, ask one focused question, not five - Flag any decision that requires my judgment before proceeding - Batch similar tasks to reduce my total number of decisions Please confirm you have absorbed this context, then ask me: what would you like to work on today?
最佳實踐
→
將此提示詞儲存為瀏覽器書籤或 Notion 快速連結頁面，確保每次都能快速貼入
→
如果使用 Claude Projects，此提示詞可直接放入 Project Instructions，之後無需重複貼
→
啟動後若 Claude 的摘要有任何錯誤，立即在第一輪修正，不要等到後面才說
→
建議每個主題方向開獨立對話（例：Life Dashboard / 國發專案 / 個人品牌），避免脈絡混亂
2
WF-2｜快速接手工作流
用於：Notion 操作中途遇到問題、想在新對話繼續上次的設計任務
目的：從中途接手，不重頭說明 關鍵：先搜尋歷史，再繼續
步驟流程
  • 先在原對話（若還開著）用 WF-2-A 提示詞「截取進度快照」
  • 複製 Claude 回覆的進度快照文字
  • 開新對話，貼入 WF-1 主啟動提示詞（必做）
  • 緊接著貼入 WF-2-B「接手提示詞」+ 剛才截取的快照
  • Claude 確認接手後，繼續工作
步驟一：在原對話截取快照 (WF-2-A)
在原對話最後貼入此提示詞Before I close this chat, generate a HANDOFF SNAPSHOT in the following format. I will paste this into the next conversation to resume instantly. ---HANDOFF SNAPSHOT START--- DATE: [today's date] TOPIC: [what we were working on] COMPLETED IN THIS SESSION: - [bullet list of what was done] CURRENT STATUS: - [exactly where we stopped] PENDING DECISIONS (need my input): - [any unresolved questions] NEXT STEP (ready to execute): - [the very next action to take] DESIGN DECISIONS MADE: - [any structural/logic decisions locked in] WARNINGS / THINGS TO AVOID: - [mistakes made or paths we ruled out] ---HANDOFF SNAPSHOT END---
步驟二：在新對話接手 (WF-2-B)
貼完 WF-1 後，接著貼入此提示詞 + 快照I'm continuing work from a previous session. Below is the handoff snapshot from that session. Please read it carefully, confirm you understand where we left off, then proceed with the NEXT STEP listed — or flag if you need any clarification before acting. [PASTE YOUR HANDOFF SNAPSHOT HERE]
最佳實踐
→
若原對話已關閉，Claude 可用「搜尋過往對話」功能找回記錄，效果等同快照
→
快照建議存在 Notion 的「工具備忘錄」頁面，作為人工備份
→
每次工作超過 45 分鐘就主動截取一次快照，防止對話過長導致遺忘
3
WF-3｜新筆記整合工作流
用於：新增筆記、新建任務卡片後，要同步更新到 Life Dashboard
目的：保持 Dashboard 與筆記同步 觸發時機：每週一次或有重大新增時
步驟流程
  • 確認本次新增或修改的 Notion 內容（頁面名稱 / 資料庫 / 欄位）
  • 開新對話或在當前對話貼入 WF-3 整合提示詞
  • 告訴 Claude 新增了什麼、在哪裡
  • Claude 掃描後提出「應更新 Dashboard 的哪些區塊」，你確認
  • Claude 執行更新，你做最終確認
整合提示詞 (WF-3-A)
每次有新筆記要整合時使用I've added or updated the following in Notion. Please help me integrate it into my Life Dashboard without creating new friction. NEW OR UPDATED CONTENT: [Describe what you added — e.g., "A new task card for 國發申請 A01-32", "Updated my health tracking page", "Added a new external file index entry for Q1 investor deck"] INTEGRATION TASK: 1. Identify which section(s) of the Life Dashboard this affects 2. Propose the minimal change needed to surface this information at the right level 3. Check if any existing Dashboard widgets, views, or links are now outdated 4. If any external files are involved, add them to the External File Index with: File name | Location path | Last updated | Related Notion page DESIGN CONSTRAINT: Do not create new structure unless absolutely necessary. Reuse existing sections, views, and links wherever possible. Please propose the changes before executing.
外部檔案索引標準格式 (WF-3-B)
建立外部檔案索引時使用此欄位模板External File Index entry format: | File Name | File Type | Storage Location | Last Updated | Related Notion Page | Notes | |-----------|-----------|-----------------|--------------|---------------------|-------| | [name] | [xlsx/pptx/pdf/paper] | [Google Drive path / local folder / physical location] | [YYYY-MM-DD] | [Notion page link] | [any notes] |
最佳實踐
→
不要等到積累很多新增後才整合，週頻率剛好 — 積太多容易出錯
→
外部檔案（PPT/Excel/紙本）只需在 Notion 登錄索引，不需要移動或轉換格式
→
整合時 Claude 應該問「這個要出現在哪個生活維度？」確保六維覆蓋
4
WF-4｜協作脈絡還原工作流
用於：過了很長時間回來、想知道之前做過什麼、避免重複犯錯
目的：還原設計決策與錯誤教訓 關鍵：先讓 Claude 搜尋歷史
步驟流程
  • 貼入 WF-1 主啟動提示詞
  • 貼入 WF-4 脈絡還原提示詞，Claude 會自動搜尋過往對話
  • Claude 生成「協作歷史摘要」，包含已完成事項、設計決策、已知錯誤
  • 你確認摘要是否準確，補充 Claude 可能遺漏的部分
  • 依據摘要決定今天從哪裡繼續
脈絡還原提示詞 (WF-4-A)
久未使用後回來時貼入I'm returning after a break and need to rebuild full context before continuing. Please search our past conversations and generate a COLLABORATION HISTORY SUMMARY covering: 1. COMPLETED WORK — What has already been done to my Notion system? List by category (structure, dashboard, external files, review mechanism, specific projects) 2. LOCKED DESIGN DECISIONS — What structural or logic decisions did we finalize? (These should not be reopened without good reason) 3. KNOWN MISTAKES & DEAD ENDS — What approaches did we try that didn't work? What should we avoid repeating? 4. OUTSTANDING ITEMS — What was left unfinished or pending my decision? 5. CURRENT SYSTEM STATE — Describe the current state of my Life Dashboard and PKM system as you understand it After generating this summary, ask me: "Does this match your recollection? What should I correct or add?" Then we can decide where to pick up from.
錯誤教訓登記格式 (WF-4-B)
當確認某個方法不適用時，要求 Claude 記錄Please add this to our "lessons learned" record for this project: MISTAKE / DEAD END: [describe what didn't work] CONTEXT: [why we tried it] CORRECT APPROACH: [what works instead] PRINCIPLE: [the underlying rule to remember] Keep this in your active context for the rest of this session, and include it in any future handoff snapshots.
最佳實踐
→
每隔 2-3 週建議主動執行一次 WF-4，即使沒有遇到問題 — 定期校正方向
→
確認 Claude 的摘要後，把「已鎖定的設計決策」存到 Notion 工具備忘錄，不要只靠 Claude 記憶
→
若 Claude 對某個已決定的事項提出新建議，先問「我們之前是否討論過這個？」
5
WF-5｜自動化更新工作流
用於：每週定期維護、批次更新 Dashboard、最小化手動操作
目的：讓更新變成例行、低阻力 觸發時機：每週日 09:00 固定 15 分鐘
步驟流程
  • 每週日開啟一個固定的「每週維護對話」（建議用同一個 Project）
  • 貼入 WF-5-A「週維護掃描提示詞」
  • Claude 掃描 Notion，列出需要更新的項目
  • 你花 5 分鐘確認清單，勾選「執行 / 跳過 / 稍後」
  • Claude 批次執行你確認的更新
  • Claude 輸出本週更新摘要，你存到 Notion 週報頁面
週維護掃描提示詞 (WF-5-A)
每週日貼入此提示詞Weekly Notion maintenance scan. Today is [DATE]. Please do the following: SCAN: 1. Check my Life Dashboard — are all six life dimensions (company, family, wealth, health, learning, relationships) still accurately represented? 2. Check the External File Index — any entries with "Last Updated" older than 2 weeks that need review? 3. Check all active task cards — identify any that are overdue, blocked, or have empty required fields 4. Check the review mechanism — was last week's review completed? If not, flag it. OUTPUT FORMAT: Generate a prioritized update list in this format: [ ] CRITICAL — [item] | Action needed: [specific action] [ ] NORMAL — [item] | Action needed: [specific action] [ ] OPTIONAL — [item] | Action needed: [specific action] After I mark which items to act on, execute them in a single batch. Confirm when done.
最小摩擦力回顧提示詞 (WF-5-B)
用於週回顧 — 設計為「5 分鐘能完成」Run my minimal weekly review. I have 5 minutes. Go fast. FORMAT: For each life dimension, give me ONE line: Company: [status in 10 words or less] Family: [status in 10 words or less] Wealth: [status in 10 words or less] Health: [status in 10 words or less] Learning: [status in 10 words or less] Relationships: [status in 10 words or less] Then give me: TOP PRIORITY THIS WEEK: [one thing only] ONE THING TO CAPTURE: [any loose information that should be recorded] ONE THING TO DELEGATE: [any task that shouldn't stay on my plate] Done. That's the whole review.
最佳實踐
→
Google Calendar 已設定每週日 09:00 提醒，這是 WF-5 持續運作的唯一觸發點
→
若某週真的太忙，用 WF-5-B 的 5 分鐘版本代替，寧願做小的也不要跳過
→
Notion 的 Automations 功能可設定「每週自動建立週報頁面」，降低手動成本
可重用技能模組 (Skill Module)

此工具包可結構化為一個可重用的「PKM 協作技能模組」，存放在 Claude Projects 的 Project Instructions 中，讓每次新對話自動載入。以下是建議的模組架構：

IDENTITY MODULE
WF-1-A 全域背景脈絡，每次對話必載
HANDOFF MODULE
WF-2-A/B 快照格式，中途接手用
INTEGRATION MODULE
WF-3-A 新筆記整合標準流程
HISTORY MODULE
WF-4-A 脈絡還原與教訓記錄
MAINTENANCE MODULE
WF-5-A/B 週維護與最小回顧
FILE INDEX MODULE
外部檔案索引標準欄位格式
存放建議：Notion 工具備忘錄頁面結構
  • 工具備忘錄（Notion 子頁面，掛在 Life Dashboard 下）✅ 已建立
  • 子頁面 1：WF-1 提示詞（全域啟動）✅ 已建立
  • 子頁面 2：WF-2 提示詞（快速接手）✅ 已建立
  • 子頁面 3：WF-3 提示詞（新筆記整合）✅ 已建立
  • 子頁面 4：WF-4 提示詞（脈絡還原）✅ 已建立
  • 子頁面 5：WF-5 提示詞（週維護）✅ 已建立
  • 子頁面 6：設計決策記錄 ✅ 已建立（含 3 條核心決策）
  • 子頁面 7：錯誤教訓記錄 ✅ 已建立（含 2 條確認教訓）
決策節點標準格式
每次鎖定設計決策時，要求 Claude 用此格式記錄DESIGN DECISION LOG entry: DATE: [YYYY-MM-DD] DECISION: [what was decided] RATIONALE: [why — key reasoning] ALTERNATIVES REJECTED: [what else was considered and why rejected] AFFECTS: [which parts of the system this changes] DO NOT REOPEN unless: [condition that would warrant revisiting]
技能模組使用說明

建議做法：將 WF-1-A 全文放入 Claude Projects 的「Project Instructions」欄位，這樣所有在該 Project 下開啟的對話都會自動載入，無需每次手動貼入。其他 WF 的提示詞則依需要在對話中貼入。

更新節奏建議：每季度（約每 12 週）回顧一次工具包本身的設計，根據實際使用中的摩擦點調整提示詞內容。
